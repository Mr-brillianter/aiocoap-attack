# CoAP 攻击流量捕获与数据集构建方法论

## 1. 研究背景与目标

鉴于当前针对受限应用协议 (Constrained Application Protocol, CoAP) 的公开威胁数据集存在显著缺失，特别是缺乏完整的数据包级捕获 (full packet capture) 数据，本研究设计并实施了一套系统化的实验方法论，旨在通过模拟典型的 CoAP 网络攻击场景并利用 Wireshark 进行全流量捕获，从而构建一个包含正常流量基线与多种攻击模式的标注数据集，以支持基于机器学习的威胁检测系统研发。本方法论涵盖攻击场景设计、流量捕获、数据标注和数据集验证四个核心阶段，重点关注两类典型攻击：重放攻击 (Replay Attack) 和拒绝服务攻击 (Denial of Service, DoS)，通过在可控实验环境中生成具有明确标签的 PCAP 格式数据集，为后续的异常检测算法开发、入侵检测系统训练以及网络安全研究提供高质量的数据基础。

## 2. 实验环境配置

本研究构建了一个基于 Windows 11 操作系统的受控实验环境，采用 Python 3.11 作为开发平台，利用 aiocoap 0.4.17 库实现 CoAP 协议栈的客户端与服务器功能，并使用 Wireshark 4.0 作为网络流量捕获与分析工具。实验网络拓扑采用本地回环接口 (localhost) 配置，通过 UDP 协议在标准 CoAP 端口 5683 上进行通信，其中攻击客户端模拟恶意行为向目标 CoAP 服务器发起请求，而 Wireshark 作为被动监听节点对所有经过该端口的数据包进行完整捕获，从而确保实验的可重复性与数据的完整性，同时避免对生产环境造成影响。

## 3. 攻击模拟方法

### 3.1 重放攻击模拟

重放攻击通过捕获合法 CoAP 请求并以高频率重复发送的方式实现，其核心特征在于保持原始请求的 Token 值不变，从而模拟攻击者试图绕过身份验证或触发重复操作的恶意行为。本研究实现的重放攻击模拟器首先向目标服务器发送一次正常请求并捕获其完整的消息结构（包括 Token、Message ID 和 Payload），随后在预设的时间窗口内（如 30 秒）以可配置的速率（如 20 请求/秒）持续重放该请求，该过程产生的流量具有显著的统计特征：Token 值重复率接近 100%、请求间隔时间极短（通常小于 100 毫秒）且请求内容完全一致，这些特征使得重放攻击流量在时序和内容层面均与正常流量存在明显差异。

### 3.2 泛洪攻击模拟

泛洪攻击通过向目标服务器发送大量并发请求以耗尽其计算资源和网络带宽的方式实现拒绝服务效果，本研究设计的泛洪攻击模拟器采用异步并发架构，能够在单一客户端上同时维持多个独立的 CoAP 会话（如 20-50 个并发连接），每个会话以高速率（如 50-100 请求/秒）向服务器发送请求，且每个请求均使用唯一的 Token 值以模拟来自不同客户端的流量。该攻击模式的关键特征包括：极高的请求速率导致单位时间内数据包数量激增、多个并发连接产生的流量在时间分布上呈现高度密集性、Token 值的高度分散性（唯一性比例超过 90%）以及可选的随机负载生成机制，这些特征共同构成了泛洪攻击在网络层面的显著指纹。

### 3.3 正常流量基线

为建立有效的对比基准，本研究同步采集正常 CoAP 通信流量作为良性样本，正常流量的生成遵循典型物联网设备的通信模式：以低频率（通常小于 5 请求/秒）、规律的时间间隔（如每 2-5 秒一次）向服务器发送传感器数据查询或状态更新请求，每个请求使用唯一的 Token 值以确保请求-响应的正确匹配，且负载大小保持在合理范围内（通常小于 100 字节）。正常流量在统计特征上表现为：Token 唯一性比例接近 100%、请求间隔时间服从规律分布、数据包到达率稳定且远低于攻击流量，这些特征为后续的异常检测模型提供了清晰的良性行为模板。

## 4. 基于 Wireshark 的流量捕获方法

### 4.1 捕获配置

流量捕获采用 Wireshark 作为主要工具，通过在网络接口层面进行被动监听实现对所有 CoAP 通信的完整记录。在 Windows 环境下，选择"Adapter for loopback traffic capture"作为监听接口以捕获本地回环流量；在捕获开始前，设置捕获过滤器 (Capture Filter) 为 `udp port 5683`，该过滤器确保仅捕获 CoAP 协议相关的 UDP 数据包，从而有效减少无关流量的干扰并降低存储开销。捕获过程中，Wireshark 将所有符合过滤条件的数据包按时间顺序完整记录，包括 UDP 头部、CoAP 协议字段（如 Token、Message ID、Code、Options）以及应用层负载，最终以 PCAP 格式保存，该格式作为网络分析领域的事实标准，具有良好的工具兼容性和可扩展性。

### 4.2 实验流程设计

数据捕获实验采用三阶段并行执行的方式进行：首先在终端 1 启动 Wireshark 并配置过滤器开始捕获，随后在终端 2 启动 CoAP 测试服务器以提供攻击目标，最后在终端 3 执行相应的流量生成程序（正常客户端或攻击脚本）。对于正常流量采集，通过 aiocoap 命令行客户端以 2-5 秒的间隔重复发送 10-20 次请求，预期捕获 20-40 个数据包（包含请求与响应），该流量呈现低频率和规律间隔的特征。对于重放攻击流量采集，执行重放攻击脚本并设置持续时间为 30 秒、请求速率为 20 请求/秒，预期捕获超过 600 个数据包，其显著特征为 Token 值的高度重复性和请求的高频率性。对于泛洪攻击流量采集，执行泛洪攻击脚本并设置持续时间为 30 秒、请求速率为 50 请求/秒、并发连接数为 20，预期捕获超过 1500 个数据包，其特征表现为 Token 值的高度分散性、极高的请求频率以及多个并发会话的交织。每次实验结束后，停止 Wireshark 捕获并将数据保存为独立的 PCAP 文件，文件命名遵循 `<流量类型>_<参数配置>_<时间戳>.pcap` 的规范以便后续管理和分析。

## 5. 数据集构建 (Dataset Construction)

### 5.1 数据集结构

```
CoAP_Attack_Dataset/
├── normal/
│   ├── normal_traffic.pcap
├── replay_attack/
│   ├── replay.pcap    
├── flood_attack/
│   ├── flood_low_rate.pcap       # 20 req/s
│   ├── flood_high_rate.pcap      # 100 req/s
└── README.md
```

**标签定义**:
- 0: 正常流量 (Normal)
- 1: 重放攻击 (Replay Attack)
- 2: 泛洪攻击 (Flood Attack)

## 8. 结论 (Conclusion)

本文档提出了一种系统化的 CoAP 攻击流量捕获方法，包括：

1. **可重复的实验环境**: 使用标准化的工具和配置
2. **多样化的攻击场景**: 涵盖重放攻击和泛洪攻击
3. **标准化的数据格式**: PCAP 格式便于后续分析
4. **详细的元数据标注**: 支持监督学习

该数据集可用于：
- 入侵检测系统 (IDS) 开发
- 机器学习模型训练
- 异常检测算法验证
- 网络安全研究
