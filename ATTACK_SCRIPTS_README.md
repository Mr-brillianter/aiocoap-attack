# CoAP 攻击脚本使用指南

本项目包含两个基于 aiocoap 库的 CoAP 攻击测试脚本，用于安全测试和研究目的。

## ⚠️ 免责声明

**这些脚本仅用于教育和授权的安全测试目的。未经授权对他人系统进行攻击是违法行为。使用者需对自己的行为负责。**

## 📋 脚本列表

### 1. CoAP 重放攻击 (`coap_replay_attack.py`)

重放攻击通过捕获合法的 CoAP 请求并重复发送，测试服务器是否能正确处理重复的请求。

**特点:**
- 自动捕获原始请求
- 使用相同的 token 进行重放
- 可配置攻击持续时间和速率
- 内置安全限制防止系统崩溃

**使用方法:**

```bash
# 基本用法
python coap_replay_attack.py coap://localhost/sensor

# 指定持续时间和速率
python coap_replay_attack.py coap://localhost/sensor --duration 10 --rate 5

# 使用 POST 方法并携带负载
python coap_replay_attack.py coap://localhost/data -m POST -p "test data" -d 20 -r 10
```

**参数说明:**
- `target`: 目标 CoAP URI（必需）
- `-m, --method`: CoAP 方法 (GET/POST/PUT/DELETE)，默认 GET
- `-p, --payload`: 请求负载内容
- `-d, --duration`: 攻击持续时间（秒），默认 30，最大 300
- `-r, --rate`: 每秒请求数，默认 10，最大 100

**示例输出:**

### 2. CoAP 泛洪攻击 (`coap_flood_attack.py`)

泛洪攻击通过向目标服务器发送大量请求来测试其负载能力和抗 DoS 能力。

**特点:**
- 高速率请求发送
- 可配置并发连接数
- 支持随机负载生成
- 实时统计成功/失败率
- 内置安全限制和速率控制

**使用方法:**

```bash
# 基本用法
python coap_flood_attack.py coap://localhost/sensor

# 高速率攻击
python coap_flood_attack.py coap://localhost/sensor --duration 60 --rate 50 --concurrent 20

# 使用随机负载的 POST 攻击
python coap_flood_attack.py coap://localhost/data -m POST --random-payload --payload-size 500
```

**参数说明:**
- `target`: 目标 CoAP URI（必需）
- `-m, --method`: CoAP 方法 (GET/POST/PUT/DELETE)，默认 GET
- `-d, --duration`: 攻击持续时间（秒），默认 30，最大 300
- `-r, --rate`: 每秒请求数，默认 20，最大 200
- `-c, --concurrent`: 最大并发请求数，默认 10，最大 50
- `--random-payload`: 使用随机生成的负载
- `--payload-size`: 负载大小（字节），默认 100，最大 1024

**示例输出:**

## 🔧 安装依赖

```bash
# 安装 aiocoap（不包含 DTLS，适用于 Windows）
uv pip install aiocoap

# 或者安装带有其他功能的版本
uv pip install "aiocoap[oscore,prettyprint,linkheader,ws]"
```

## 🎯 测试目标设置

在进行攻击测试前，你需要一个 CoAP 服务器作为目标。可以使用 aiocoap 自带的示例服务器：

```bash
# 启动简单的 CoAP 服务器
python -m aiocoap.cli.fileserver

# 或使用项目中的示例服务器
python serverGET.py
```

## 📊 安全限制

为了防止意外造成系统崩溃，脚本内置了以下安全限制：

| 参数 | 最大值 | 说明 |
|------|--------|------|
| 持续时间 | 300 秒 (5分钟) | 防止长时间攻击 |
| 重放速率 | 100 请求/秒 | 重放攻击速率限制 |
| 泛洪速率 | 200 请求/秒 | 泛洪攻击速率限制 |
| 并发数 | 50 | 最大并发连接数 |
| 负载大小 | 1024 字节 | 单个请求最大负载 |

## 🛡️ 防御建议

如果你是服务器管理员，可以采取以下措施防御这些攻击：

1. **重放攻击防御:**
   - 实现 token 唯一性检查
   - 使用时间戳验证请求新鲜度
   - 启用 OSCORE 安全协议

2. **泛洪攻击防御:**
   - 实施速率限制（Rate Limiting）
   - 使用防火墙规则限制请求频率
   - 部署负载均衡器
   - 启用 DTLS 进行身份验证

## 📝 注意事项

1. **仅在授权环境中使用** - 确保你有权限测试目标系统
2. **监控系统资源** - 攻击过程中注意监控 CPU、内存和网络使用情况
3. **逐步增加强度** - 从低速率开始，逐步增加以观察系统行为
4. **保存测试日志** - 记录测试结果用于分析和改进

## 🔍 故障排除

**问题: 所有请求都失败**
- 检查目标服务器是否运行
- 确认 URI 格式正确（例如：`coap://localhost:5683/path`）
- 检查防火墙设置

**问题: 成功率很低**
- 降低请求速率
- 减少并发连接数
- 增加超时时间（需修改脚本）

**问题: 脚本运行缓慢**
- 检查网络延迟
- 降低并发数
- 确保目标服务器有足够资源
