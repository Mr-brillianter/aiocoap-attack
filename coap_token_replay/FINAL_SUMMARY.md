# CoAP Token 重用攻击 - MITM 实现总结

## 📋 问题分析

### 问题 1: 客户端收到了第一个响应

**原因**: 使用 aiocoap 库时，客户端可能会直接连接到服务器，绕过攻击者。

**解决方案**: 使用原始 socket 发送 CoAP 请求，确保流量通过攻击者。

### 问题 2: logger.info() 错误

**原因**: `logger.info()` 需要一个消息参数。

**解决方案**: 使用 `logger.info("")` 而不是 `logger.info()`。

### 问题 3: 4.04 Not Found

**原因**: 服务器没有 `/time` 路径。

**解决方案**: 在 `server.py` 中添加 `TimeResource` 类。
