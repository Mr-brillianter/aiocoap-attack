# CoAP Token 重用攻击 - MITM 实现指南

## 📋 概述

这是一个完整的 CoAP Token 重用攻击实现，使用**中间人代理（MITM）**架构。

### 核心思想

让攻击者成为客户端和服务器之间的**必经之路**，从而可以：
- 拦截所有通信
- 缓存特定响应
- 延迟转发响应
- 重放旧响应

## 🏗️ 架构设计

```
┌─────────┐         ┌──────────┐         ┌─────────┐
│ Client  │────────▶│ Attacker │────────▶│ Server  │
│ :random │         │  :5684   │         │  :5683  │
└─────────┘         └──────────┘         └─────────┘
     ▲                    │                    │
     │                    │                    │
     └────────────────────┘                    │
          (延迟重放)                            │
                                               │
                          ◀────────────────────┘
```

### 端口分配

| 角色 | 地址 | 说明 |
|------|------|------|
| **Server** | `127.0.0.1:5683` | 真实的 CoAP 服务器 |
| **Attacker** | `127.0.0.1:5684` | MITM 代理（监听此端口） |
| **Client** | `127.0.0.1:random` | 客户端（连接到攻击者） |

## 🎯 攻击流程

### 1. 正常流程（无攻击）

```
Client → Server: 请求 A (Token: T)
Server → Client: 响应 A' (Token: T)
```

### 2. 攻击流程（MITM）

```
时刻 T0: Client → Attacker: 请求 A (Token: T)
时刻 T1: Attacker → Server: 转发请求 A
时刻 T2: Server → Attacker: 响应 A' (Token: T)
时刻 T3: Attacker: 缓存 A'，不转发！
时刻 T4: Client: 超时，重试
时刻 T5: Client → Attacker: 请求 B (Token: T, 相同!)
时刻 T6: Attacker → Server: 转发请求 B
时刻 T7: Server → Attacker: 响应 B' (Token: T)
时刻 T8: Attacker → Client: 转发 B'
时刻 T9: Client: 处理 B'
时刻 T10: Attacker → Client: 重放 A' (Token: T)
时刻 T11: Client: 误认为 A' 是新响应！
```

  --listen-host 127.0.0.1 \
  --listen-port 5684 \
  --server-host 127.0.0.1 \
  --server-port 5683 \

## 🐛 故障排除

### 问题 1: 端口已被占用

```
OSError: [WinError 10048] 通常每个套接字地址(协议/网络地址/端口)只允许使用一次。
```

**解决方案**：
- 检查是否有其他程序占用端口
- 使用 `--listen-port` 参数指定其他端口

### 问题 2: 客户端连接超时

**解决方案**：
- 确保服务器已启动
- 确保攻击者已启动
- 检查防火墙设置
